# Cohesion and Coherence Diagnostics

Use this reference when a draft feels disjointed, when paragraphs do not build an argument, or when the user asks for reader-friendly scientific writing.

## Core Distinction

- Cohesion is local flow: each sentence connects to what came before and prepares what comes next.
- Coherence is global unity: the paragraph, section, or article has a governing point the reader can recognize.
- A passage can be cohesive but not coherent if each sentence links locally while the topic keeps drifting.

## Reader-First Framework

Use these five checks during drafting and revision:

1. No unexplained novelty.
   - A new concept may appear only if it was named earlier, implied by the previous sentence, or introduced with an explicit purpose.
   - If the reader must ask "why are we talking about this now?", add a bridge or reorder.

2. One paragraph equals one reader question.
   - Before revising a paragraph, write the question the reader has at that point.
   - The topic sentence should answer, frame, or narrow that question.
   - Sentences that answer another question usually need a new paragraph.

3. Old-to-new progression.
   - Start sentences with familiar information the reader already has access to.
   - End sentences with new, complex, emphasized, or consequential information.
   - Let the new information at the end of one sentence become the old information near the start of the next.

4. Signpost structural changes.
   - Signal topic shifts, contrasts, causes, consequences, examples, exceptions, and moves from result to interpretation.
   - Use signposts to reveal logic, not to decorate weak organization.

5. Reader memory discipline.
   - Assume the reader remembers the previous paragraph more reliably than earlier details.
   - Repeat key terms before introducing synonyms.
   - Define one technical concept at a time unless writing for readers who already know the vocabulary.

## Paragraph Audit

For each paragraph, check:

- Governing idea: Can the main point be stated in one sentence?
- Reader question: What question does this paragraph answer for this reader?
- Topic sentence: Does the first or second sentence orient the reader?
- Development: Do the following sentences explain, support, qualify, compare, exemplify, or apply the topic sentence?
- Unity: Does any sentence serve a different governing idea?
- Exit: Does the final sentence leave the reader ready for the next paragraph?

## Generated-Article Self-Test

Before returning a drafted scientific article, silently test four things:

- Coherence: Can the article's central claim be stated in one sentence, and does every section serve that claim?
- No unexplained novelty: Does every new term, variable, subgroup, result, limitation, or implication have a reason to appear when it appears?
- Old-to-new flow: Do sentence openings usually connect to familiar information, with new or heavier information placed later?
- Paragraph question discipline: Can each paragraph be paired with one reader question without forcing two unrelated questions together?

If any answer is no, revise the order or add a bridge before polishing style.

## Sentence Flow Audit

Use this when flow feels broken:

1. Underline or list the first five to six words of each sentence.
2. Check whether those openings form a related set of topics.
3. Mark sentence openings that introduce new concepts without preparation.
4. Move familiar context to the beginning and heavier new information toward the end.
5. Repeat a key noun or use an unambiguous pronoun when the reader needs a thread.

## Common Fixes

- Move a buried topic sentence to the beginning of the paragraph.
- Split a paragraph that answers two reader questions.
- Add a bridge sentence before a new concept, method, population, mechanism, or implication.
- Replace vague pronouns such as "this" with named concepts when ambiguity is possible.
- Repeat the same technical term until the reader has a stable mental model.
- Reorder sentences so evidence appears before interpretation when the reader needs proof.
- Reorder sentences so interpretation appears before details when the reader first needs the point.
